Crea y gestiona compras, ofertas y pedidos.

US-1Gestión de compras

Crear una compra

Como organizador, quiero crear una compra empezando por las ofertas y completar las fechas cuando las tenga, para poder armarla aunque todavía no tenga el cronograma cerrado.

El wizard va: ofertas → cupo, retiro y reparto (% para el mercado, el resto a proveedores) → fechas. Las fechas son opcionales: sin fecha de entrega la compra queda como borrador (no visible al público) hasta definirla.

US-2Gestión de compras

Definir y actualizar el lugar de retiro

Como organizador, quiero definir o cambiar el lugar de retiro en cualquier momento, incluso después de crear la compra, para poder anunciarlo aunque al principio no lo sepa.

Puede estar vacío; cuando se define se muestra públicamente; al editarlo se actualiza la vista pública y las páginas de estado del comprador.

US-3Gestión de compras

Definir un cupo de participantes

Como organizador, quiero limitar cuántas personas pueden pedir en una ventana, con un valor por defecto razonable, para no tomar más pedidos de los que puedo manejar.

El cupo es editable; se aplica un valor por defecto si no se define.

US-4Gestión de compras

Agregar ofertas

Como organizador, quiero agregar ofertas a una compra con nombre, descripción, precio y stock, para que los compradores sepan qué hay disponible.

No requiere foto ni unidad de medida; cada oferta tiene un máximo por comprador con un valor por defecto razonable.

US-10Procesar pedidos

Revisar pedidos

Como organizador, quiero ver todos los pedidos de una compra con los datos de contacto del comprador y su estado, para poder procesarlos.

US-11Procesar pedidos

Confirmar un pedido

Como organizador, quiero marcar un pedido como confirmado después de ver el comprobante de pago por WhatsApp, para que el comprador sepa que está listo.

Confirmar pasa el pedido de pendiente → confirmado y dispara el email de confirmación.

US-12Procesar pedidos

Cancelar un pedido

Como organizador, quiero cancelar un pedido pendiente que nunca se pagó, para liberar sus unidades y su lugar en el cupo para otros.

Cancelar libera stock + lugar; el estado del comprador muestra "cancelado"; no hay expiración automática.

US-13Procesar pedidos

Exportar pedidos

Como organizador, quiero descargar los pedidos de una compra como CSV, para organizar el retiro y la entrega.

US-15Cierre

Liquidar una compra

Como organizador, quiero cerrar las cuentas de una compra entregada, para pagar a proveedores y saber la ganancia del mercado.

Sobre los pedidos confirmados: total vendido, monto a proveedores y ganancia del mercado (% de la compra, menos el transporte que se carga acá), con desglose por oferta. "Marcar liquidada" deja la compra en estado liquidada.

US-5Comprar

Explorar una compra abierta

Como comprador, quiero ver las ofertas de una compra con precio, stock y cuántas ya se compraron, para decidir qué pedir.

Cada oferta muestra dos contadores — confirmadas y pendientes (ej. "12 confirmadas, 5 pendientes"); retiro + fecha de entrega se muestran cuando están definidos.

US-6Comprar

Hacer un pedido con carrito

Como comprador, quiero agregar varias ofertas a un carrito y finalizar la compra una sola vez con mi nombre, apellido, email y teléfono, para pedir todo en un solo paso.

Solo mientras la ventana está abierta; el pedido se crea como pendiente; respeta los máximos por comprador.

US-7Comprar

Ver mensajes de "sin stock" / "cupo completo"

Como comprador, quiero un mensaje claro cuando algo no se puede pedir, para entender por qué.

"Sin stock" cuando se agotan las unidades de una oferta; "cupo completo" cuando se alcanza el cupo de participantes; ambos cuentan todos los pedidos realizados (pendientes + confirmados).

US-8Comprar

Seguir mi pedido

Como comprador, quiero un enlace único (también por email) para ver el estado de mi pedido, para seguirlo sin necesidad de una cuenta.

El enlace muestra el estado actual; después de cerrarse la ventana muestra el lugar de retiro + la fecha de entrega.

US-9Comprar

Recibir notificaciones

Como comprador, quiero un email cuando hago el pedido y cuando se confirma, para saber qué está pasando.

El email de pedido incluye el enlace de estado; el email de confirmación incluye los datos de retiro cuando están disponibles.

US-14Ciclo de la ventana

Los pedidos abren y cierran por tiempo

Como comprador, quiero que solo se pueda pedir durante la ventana, para que el proceso sea justo y predecible.

Borrador (sin fechas, no visible al público) → próximo (antes del inicio) → abierto (entre inicio y fin, se puede pedir) → cerrado (después del fin: pedidos deshabilitados, se destacan retiro + entrega) → liquidada (cuentas cerradas y proveedores pagados).
