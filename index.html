Crea y gestiona el catálogo, las compras y los pedidos.

US-1Gestión de compras

Crear una compra

Como organizador, quiero crear una compra —ventana, retiro y % del mercado— para agrupar los pedidos; los productos salen del catálogo, no se cargan acá.

La compra solo agrupa pedidos: no se eligen productos (al presentarla muestra todos los del catálogo con stock). El wizard va: retiro y reparto (% para el mercado, el resto a proveedores) → fechas. Las fechas son opcionales: sin fecha de entrega queda como borrador (no visible al público). Solo puede haber una compra abierta a la vez.

US-2Gestión de compras

Definir y actualizar el lugar de retiro

Como organizador, quiero definir o cambiar el lugar de retiro en cualquier momento, incluso después de crear la compra, para poder anunciarlo aunque al principio no lo sepa.

Puede estar vacío; cuando se define se muestra públicamente; al editarlo se actualiza la vista pública y las páginas de estado del comprador.

US-3Gestión de compras

Compartir el link de la compra

Como organizador, quiero un link de la compra para compartir, para enviarlo solo a ciertas personas cuando quiero limitar quién participa.

No hay cupo numérico: se limita compartiendo el link (ej. por WhatsApp) con quienes uno quiera. Cualquiera con el link puede pedir mientras la ventana esté abierta.

US-16Catálogo

Administrar catálogo

Como organizador, quiero mantener el catálogo de productos y su stock actual, para que las compras muestren siempre lo disponible.

Los productos son independientes de las compras. Se recorren con flechas ‹ › o ←/→; la barra espaciadora abre el editor para actualizar stock, precio y unidad de medida. Al costado, todos los productos con su stock (alfabético, paginado).

US-10Procesar pedidos

Revisar pedidos

Como organizador, quiero ver todos los pedidos de una compra con los datos de contacto del comprador y su estado, para poder procesarlos.

US-11Procesar pedidos

Confirmar un pedido

Como organizador, quiero marcar un pedido como confirmado después de ver el comprobante de pago por WhatsApp, para que el comprador sepa que está listo.

Confirmar pasa el pedido de pendiente → confirmado y dispara el email de confirmación.

US-12Procesar pedidos

Cancelar un pedido

Como organizador, quiero cancelar un pedido pendiente que nunca se pagó, para liberar sus unidades para otros.

Cancelar libera el stock; el estado del comprador muestra "cancelado"; no hay expiración automática.

US-13Procesar pedidos

Exportar pedidos

Como organizador, quiero descargar los pedidos de una compra como CSV, para organizar el retiro y la entrega.

US-15Cierre

Liquidar una compra

Como organizador, quiero cerrar las cuentas de una compra entregada, para pagar a proveedores y saber la ganancia del mercado.

Sobre los pedidos confirmados: total vendido, monto a proveedores y ganancia del mercado (% de la compra, menos el transporte que se carga acá), con desglose por producto. "Marcar liquidada" deja la compra en estado liquidada.

US-5Comprar

Explorar una compra abierta

Como comprador, quiero ver los productos disponibles en una compra con precio, stock y cuántas ya se compraron, para decidir qué pedir.

Muestra todos los productos del catálogo con stock (vivo). Cada uno con precio, stock y dos contadores — confirmadas y pendientes (ej. "12 confirmadas, 5 pendientes"); retiro + fecha de entrega cuando están definidos.

US-6Comprar

Hacer un pedido con carrito

Como comprador, quiero agregar varios productos a un carrito y finalizar la compra una sola vez con mi nombre, apellido, email y teléfono, para pedir todo en un solo paso.

Solo mientras la ventana está abierta; el pedido se crea como pendiente.

US-7Comprar

Ver mensaje de "sin stock"

Como comprador, quiero un mensaje claro cuando algo no se puede pedir, para entender por qué.

"Sin stock" cuando se agota el stock de un producto, contando todos los pedidos realizados (pendientes + confirmados).

US-8Comprar

Seguir mi pedido

Como comprador, quiero un enlace único (también por email) para ver el estado de mi pedido, para seguirlo sin necesidad de una cuenta.

El enlace muestra el estado actual; después de cerrarse la ventana muestra el lugar de retiro + la fecha de entrega.

US-9Comprar

Recibir notificaciones

Como comprador, quiero un email cuando hago el pedido y cuando se confirma, para saber qué está pasando.

El email de pedido incluye el enlace de estado; el email de confirmación incluye los datos de retiro cuando están disponibles.

US-14Ciclo de la ventana

Los pedidos abren y cierran por tiempo

Como comprador, quiero que solo se pueda pedir durante la ventana, para que el proceso sea justo y predecible.

Borrador (sin fechas, no visible al público) → próximo (antes del inicio) → abierto (entre inicio y fin, se puede pedir) → cerrado (después del fin: pedidos deshabilitados, se destacan retiro + entrega) → liquidada (cuentas cerradas y proveedores pagados). Solo una compra abierta a la vez.

Fuera de alcance (por ahora)

Múltiples vendedores Fotos de producto Cupo de participantes Máx por comprador Pago online Subida de comprobante en la app Cuentas de comprador Cancelación por el comprador Expiración automática de pendientes

Supuesto a confirmar: los compradores no pueden editar ni cancelar sus propios pedidos — solo el admin.
